## 2. Главный флоу заявки (обмен валют)

> Это **активный канал** заявок через PuzzleBot (не через Mini App). Клиент проходит весь путь кнопками внутри чата.

### Точка входа
- Блок `Обменять валюту` (после старта/меню)

### Шаги флоу

#### Шаг 1: Выбор продаваемой валюты
Клиент выбирает что **отдаёт**:
- `RUB (безнал)` / `USD (нал)` / `USDT` / `EUR (нал)` / `VND` / `Другая валюта`

#### Шаг 2: Выбор получаемой валюты
Через блок `Продать/получить` → клиент выбирает что **получит**.

#### Шаг 3: Расчёт курса
Используется **матрица 1**: `RUB → USD 1`, `USD → RUB 1`, ..., `VND → EUR 1` (25 пар).

Связанные блоки:
- `1 Рассчитать курс (ПРОДАЁМ РУБЛИ)` / `(ПРОДАЁМ ДОЛЛ.)` / `(ПРОДАЁМ USTD)` / `(ПРОДАЁМ ЕВРО)` / `(ПРОДАЁМ VND)`
- `2 Рассчитать курс (...)` — _уточнить, в чём отличие_
- `ОБНОВИТЬ КУРС (Продать/получить)` — кнопка обновления

#### Шаг 4: Ввод суммы
- `Знаю, сколько хочу продать` → `Делаем расчет 1` → `Обменять валюту К`
- `Знаю, сколько хочу получить` → `Делаем расчет 2` → `Обменять валюту К2`

> ⚠️ **Уточнить:** в каких именно случаях используется К vs К2 — это калькулятор или часть основного флоу?

#### Шаг 5: Выбор способа оплаты (нал / безнал)
Условные блоки:
- `Условие нал - безнал`, `Условие нал - безнал 2`, `Условие нал - безнал 3`, `Условие нал - безнал 4`
- `Усл только нал`
- `Безнал при встрече`
- `Условие безнал - нал`

> ⚠️ **Подозрительно много условных блоков под одно решение** — возможно, есть упрощение.

#### Шаг 6: Выбор места встречи / способа доставки
- `Только нал` → `Ок, обменять при встрече`
- `Приеду сам(-а)` / `Приеду сам(а) менеджер` / `Время (приеду сам)`
- `Указать адрес` / `Указать адрес вручную`
- `Условие место`, `Условие место 2`

#### Шаг 7: Выбор времени
- Слоты: `8-10`, `10-12`, `12-14`, `14-16`, `16-18`, `18-20`, `20-22`
- `Другой день`, `Время (другой день)`, `Другой день - изменение курса`
- `Вечерняя доставка`, `Вечерняя доставка - другой день`
- `Менее 8000р` / `Проверка 8000р`

#### Шаг 8: Реквизиты для безналичной оплаты
**Онлайн оплата:**
- `Онлайн USDT` / `Онлайн VND` / `Онлайн RUB` / `Онлайн 2`
- По способу: `По номеру карты` / `По номеру счёта` / `По номеру телефона`
- Версии для рублей: `По номеру карты RUB` / `По номеру счёта RUB` / `По номеру телефона RUB`

**Получение:**
- `VND банк на получение`, `RUB банк на получение`, `USDT банк на получение`
- `Получение ДС банк`, `Получение ДС банк из USTD`, `Получение USDT`

**USDT сети:**
- `BEP20`, `Polygon`, `Tezos`, `TRC20`, `ERC20`, `Solana`, `ByBIT UID`, `Binance Pay ID`, `Другое (менеджер)`

#### Шаг 9: Подтверждение заявки
- `Подтвердить` / `Подтвердить безнал`
- `Подтвердить самовывоз` / `Подтвердить самовывоз безнал`
- `Подтвердить другой день`
- `Ок, жду` / `Отменить заявку`

#### Шаг 10: Уведомление о заявке
- `Уведомление обмен валюты`
- `Уведомление Квитанция` / `Уведомление Квитанция 2` / `Уведомление Квитанция 3`

## 4. Mini App интеграция

### Точки касания PuzzleBot ↔ Mini App
- 🟢 **Блок `Mini app`** (зелёный, изолированный) — точка входа после отправки заявки через Mini App. Получает уведомление от Vercel API через `PUZZLEBOT_TOKEN`.
- 🔵 **`Конец заявки (Mini App)`** — блок завершения после Mini App-заявки.
- 🔵 **`Повтор заявки (Mini App)`** — повторная заявка на основе данных из Mini App.

### Связь с Vercel
- Vercel шлёт уведомления в PuzzleBot через `PUZZLEBOT_TOKEN`
- Используется для алертов @SashaSupport и группы

> _Заполнить: точная цепочка после получения уведомления от Vercel — что конкретно показывает PuzzleBot клиенту?_

## 5. Курьер / доставка / получение

### Статусы курьера (последовательность)
1. `Курьер свободен` → ветвление:
   - `Да, можно ехать`
   - `Нет, по времени`
2. `Курьер выезжает`
3. `Курьер будет у вас ~через 30 мин`
4. `Курьер подъезжает! Осталось ~5 мин`
5. `Курьер на месте`
6. Ответы клиента: `Да, встретился с курьером` / `Не могу найти курьера`

### Реквизиты для курьера
- `Банк курьер` / `Банк курьер (2)`
- `По номеру карты курьер`
- `По номеру счета курьер`
- `По номеру телефона курьер`
- `Способ получения` / `Уведомление способ получения`
- `Получение`

### Передача валют курьером
- `USDT курьер` / `VND курьер` / `RUB курьер`
- `Уведомление банк курьер`
- Условия: `Условие банк курьер`, `Условие банк курьер 2`

### Оплата при встрече
- `Я оплатил(-а) 3 (при личной встрече)`
- Блоки `BEP20 2`, `Polygon 2`, `Tezos 2`, `TRC20 2`, `ERC20 2`, `Solana 2`, `ByBIT UID 2`, `Binance Pay ID 2`, `Другое (менеджер) 2`
- `RUB`, `USDT конец заявки`, `VND конец заявки`
- `Уведомление Квитанция 3`

> ⚠️ **Старый блок:** `Курьер на месте (Старая версия)` — кандидат на удаление, проверить связи.

## 11. Условные блоки (логика)

> Все бирюзово-зелёные блоки = условные операторы / роутеры в PuzzleBot.

### Глобальные условия
- `Условие Сумма` — проверка введённой суммы
- `Условие Валюта`
- `Условие Тип обмена`
- `Условие "Введенная сумма" К` (в калькуляторе)
- `условие запомнить валюту`

### Расчёт курса (бирюзовые)
- 4 набора по 5 блоков "Рассчитать курс (ПРОДАЁМ X)":
  - `1 Рассчитать курс (...)` — _основной флоу?_
  - `2 Рассчитать курс (...)` — _обратный расчёт?_
  - `K1 Рассчитать курс (...)` — калькулятор прямой
  - `K2 Рассчитать курс (...)` — калькулятор обратный

### Покупка / Продажа
- `ПОКУПКА` / `ПРОДАЖА`
- `ПОКУПКА К` / `ПРОДАЖА К`
- `Сумма обмена ПОКУПКА` / `Сумма обмена ПОКУПКА sale`
- `Сумма обмена ПРОДАЖА` / `Сумма обмена ПРОДАЖА sale`
- `Сумма обмена ПОКУПКА К` / `Сумма обмена ПРОДАЖА К`

### Проверки сумм
- `Проверка суммы USD/USDT`
- `Проверка суммы USDT/USD`
- `Проверка суммы USDT/USDT`
- `Проверка суммы USD/USDT ПОКУПКА`
- `Проверка суммы USDT/USD ПОКУПКА`
- `Проверка суммы USD/USDT K`
- `Проверка 8000р`

### Дата / время / место
- `Условие Дата` / `Условие самовывоз`
- `Условие другой день` / `Условие другой день - другой курс`
- `Условие на повтор заявки`
